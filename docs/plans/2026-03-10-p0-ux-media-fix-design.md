# Content Analyzer P0 修复设计

日期：2026-03-10

## 目标
1. 详情页返回列表不整页刷新，保留列表体感。
2. 推文多媒体支持图片+视频并存展示。
3. 首页预取失败后，点击预览可强制重试，不被失败缓存卡死。

## 方案
- 返回逻辑：详情页优先 `router.back()`，回退失败再按 query 恢复列表。
- 多媒体：VideoPreview 改为媒体集合浏览（视频/图片切换，默认首个视频）。
- 缓存：useMediaCache 增加失败短 TTL 与 `force` 参数；预览弹窗打开时使用 force=true 重拉。

## 验证
- 返回后列表 tab/sort/page 不丢失。
- 包含图+视频的推文能看到全部媒体且可切换。
- 首次失败后再次点预览能触发重新请求并可恢复。
