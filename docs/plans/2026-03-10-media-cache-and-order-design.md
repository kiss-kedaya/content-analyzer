# Content Analyzer 媒体缓存与原始顺序设计

日期：2026-03-10

## 目标
1. 媒体顺序按后端原始顺序返回，不再默认视频优先。
2. 预览关闭后精确回到触发卡片位置。
3. 成功提取媒体后落库缓存，避免重复调用外部提取接口。
4. 保存外部提取接口成功返回的原始数据，便于未来修复解析逻辑后重放。

## 方案
- Prisma 新增 `MediaCache` 表，按 `url` 唯一缓存。
- 字段至少包含：`url`、`status`、`rawResponse`、`parsedMedia`、`lastFetchedAt`、`createdAt`、`updatedAt`。
- `/api/preview-media` 先查库，命中成功缓存则直接返回；未命中再调用外部提取接口。
- 成功调用后，保存原始响应与解析后的有序媒体数组。
- 前端预览/缩略图统一使用有序媒体数组，不再视频优先。
- 预览打开时记录触发元素，关闭时执行滚动定位 + 焦点归位。

## 验证
- 同 URL 第二次预览不再重复请求外部接口。
- 推文被删后仍可用缓存结果。
- 图片/视频顺序与后端原始顺序一致。
- 关闭预览后回到原卡片位置。
- `npm run build` 与 Prisma 同步通过。
